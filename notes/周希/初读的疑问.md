## 基础概念：什么是均摊分析？如何计算？

- **疑问**：均摊分析（amortized analysis）的准确定义是什么？它与平均情况分析（average-case analysis）有何区别？
- **三种常用方法**：
  1. **聚合分析（Aggregate Method）**：对所有 n 个操作求总代价 T(n)，均摊代价 = T(n)/n。
  2. **记账法（Accounting Method）**：为每种操作分配"存款"，廉价操作存钱，昂贵操作取钱，保证账面始终非负。
  3. **势能法（Potential Method）**：定义势能函数 Φ，均摊代价 = 实际代价 + ΔΦ。需满足 Φ ≥ 0 且初始 Φ₀ = 0。
- **待进一步确认**：论文中的下界证明用的是哪种方法？势能法是否适用于证明下界（而不仅仅是上界）？

## 参数 α 的几何扩展/收缩技术：均摊代价如何计算？

- **策略描述**：
  - 选择参数 α > 0
  - 数组满时（增长）：分配大小为 (1+α)N 的新数组，拷贝 N 个元素
  - 使用率低于 1/(1+α)² 时（收缩）：分配大小为 N/(1+α) 的新数组，拷贝 N/(1+α) 个元素

- **疑问**：增长均摊代价 (1+α)/α 和收缩均摊代价 1/α 是如何推导出来的？

- **尝试推导（待验证）**：

  **增长均摊代价**：用势能法，定义势能函数 Φ = α·(size − capacity/(1+α))（或其他形式）。
  - 每次 O(1) 插入实际代价 = 1，势能变化 = α，均摊 = 1 + α
  - 当数组满时触发扩容，实际代价 = (1+α)N（拷贝），势能从 α·(N − N/(1+α)) = α²N/(1+α) 降为 0
  - 把扩容代价均摊到此前 N 次插入中，每次承担约 (1+α)/α

  **收缩均摊代价**：类似思路。
  - 当使用率降到 1/(1+α)² 时触发收缩，实际拷贝代价 ≈ N/(1+α)
  - 均摊代价 ≈ 1/α

- **待进一步确认**：
  1. 势能函数的具体形式是什么？如何在增长和收缩之间平衡？
  2. 参数 α 的最优取值是多少？（令 (1+α)/α = 1/α 得到 α = ? 或者优化其他目标？）
  3. 这个策略与论文中提出的 Optimal Resizable Array 有何差距？

## 为什么均摊代价是 O(1)？

- **疑问**：为什么可调整数组（resizable array）的插入/删除操作可以实现均摊 O(1)？明明有时候需要 O(N) 的时间来扩容或收缩。

- **直觉理解**：
  - 扩容（倍增策略）：每次扩容时容量翻倍，N 个元素拷贝的总代价 = N + N/2 + N/4 + … ≈ 2N = O(N)，分摊到 N 次插入，每次 O(1)。
  - 收缩策略同理，关键在于扩容/收缩操作不能太频繁。
  - 核心机制：让昂贵操作之间的间隔指数级增长，使得总拷贝代价被足够多次的廉价操作所均摊。

- **待进一步确认**：
  1. 如果增长和收缩同时存在，还能保证 O(1) 吗？还是如 3.1 所说退化到 Ω(N)？
  2. HAT 或论文中提出的最优结构是如何克服这个退化问题的？
  3. O(1) 中的常数因子与空间利用率之间如何 trade-off？

## 推论 4.3：存储空间与临时空间之间的权衡

- **疑问**：推论 4.3 说"任何使用 N + O(N^(1/r)) 空间存储可调整数组的数据结构，在增长操作期间必须偶尔使用 N + Ω(N^(1−1/r)) 空间"。这句话是什么意思？为什么存储空间和临时空间之间存在这样的 trade-off？

- **原文**：论文 Section 4，推论 4.3（Corollary 4.3），作为定理 4.2（s(N) · t(N) ≥ N）的直接推论。

- **尝试拆解**：
  - **s(N)**：稳定状态下存储 N 个元素所需的**额外空间**（超出 N 本身的部分）
  - **t(N)**：增长操作期间临时需要的**额外空间**（超出当前元素数量 N 的部分）
  - **定理 4.2**：任何可调整数组实现必须满足 s(N) · t(N) ≥ N
  - 如果存储额外空间 s(N) = O(N^(1/r))（很小），代入定理 4.2 得到：
    - t(N) ≥ N / s(N) = N / O(N^(1/r)) = Ω(N^(1−1/r))
    - 即临时空间被迫很大

- **具体例子**：
  - r = 1：存储额外空间 O(N)，临时额外空间 Ω(1) — 用大量存储换少量临时空间
  - r = 2：存储额外空间 O(√N)，临时额外空间 Ω(√N) — Brodnik et al. (1999) 的结果
  - r = 3：存储额外空间 O(N^(1/3))，临时额外空间 Ω(N^(2/3)) — 存储更省，但临时空间更大

- **我没看懂的地方**：
  1. 定理 4.2 的直觉是什么？为什么 s(N) 和 t(N) 的乘积一定 ≥ N？这个乘积下界是怎么证明的？
  2. 结论中的"偶尔"（occasionally）是什么意思？是指不是每次增长都触发，但某些增长操作不可避免？
  3. 逆否命题怎么理解：如果我想让临时空间也很小（比如 O(√N)），是不是意味着存储空间至少要 Ω(√N)？这跟"r 越大存储越省但临时空间越大"的直觉是一致的吗？

- **待进一步确认**：定理 4.2 的完整证明（增长游戏模型），以及推论 4.3 与第 6 节构造性上界（O(rN^(1/r)), O(N^(1−1/r))-实现）的匹配关系。
