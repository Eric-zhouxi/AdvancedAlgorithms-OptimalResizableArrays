读Abstract & Introduction后理解：

可调整数组：
标准情况下使用O（N）空间维护大小为N的数组，每次操作平均O（1），甚至最坏O（1）
后证明使用仅 N + O(N^1/2) 空间维护大小为 N 的可调整数组，每次操作仍为 O(1) 时间
本论文证明：如果增长和收缩操作期间可以短暂使用 N + O(N^(1−1/r)) 空间，则 N + O(N^(1/r)) 空间足以存储和访问大小为 N 的数组，增长和收缩操作需要 O(r) 均摊时间。

后台重建技术：
并不是单独开一个后台线程，而是保留原数组并开新的数组，在访问和加入元素时顺带进行复制，避免大批量的一次性操作，也就是“去摊还化”。另外，论文中特别提到r=2的特殊情况，因为当r=2时，N^r==N^（1-r），平时允许的额外空间约等于重建时需要的额外空间，可以将复制分散到后续的多次操作中。r>2时，临时空间变成数据结构平时持续占用的空间，N^（1-r）> N^r
